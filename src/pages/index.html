Rentals API Documentation

Endpoints

Get Available Bikes

Method: GET

URL: /api/vehicles

Description: Retrieve the current list of available vehicles at various locations.

Response:

{
    "_id": "string",
    "type": "string",
    "station": "string",
    "available": "boolean"
}

Get Bike Details

Method: GET

URL: /api/vehicles/{vehicle_id}

Description: Retrieve detailed information about a specific bicycle/ scooter/ skateboard.

Response:

{
    "_id": "string",
    "type": "string",
    "station": "string",
    "available": "boolean"
}

Reserve a Bike

Method: POST

URL: /api/vehicles/{vehicle_id}/reserve

Description: Reserve a vehicle for a specified time period.

Request Body:

{
    "user_id": "string",
    "start_time": "string",
    "end_time": "string"
}

Response:

{
    "reservation_id": "string",
    "vehicle_id": "string",
    "user_id": "string",
    "start_time": "string",
    "end_time": "string",
    "status": "string"
}

Cancel a Reservation

Method: DELETE

URL: /api/reservations/{reservation_id}

Description: Cancel an existing bike reservation.

Response:

{
    "reservation_id": "string",
    "status": "string"
}

Get User Reservations

Method: GET

URL: /api/users/{user_id}/reservations

Description: Get a list of current and past reservations for a user.

Response:

{
    "reservations": [
        {
            "reservation_id": "string",
            "bike_id": "string",
            "start_time": "string",
            "end_time": "string",
            "status": "string"
        }
    ]
}

Error Handling

Common Error Responses:

  • 400 Bad Request: Invalid input or parameters.
  • 401 Unauthorized: User not authenticated.
  • 404 Not Found: Resource not found (e.g., bike or reservation).
  • 500 Internal Server Error: General server error.

Authentication

Authorization: Use API keys or OAuth tokens.

Example Header:

Authorization: Bearer {token}
