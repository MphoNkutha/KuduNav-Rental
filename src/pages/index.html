Vehicle Rental API Documentation

This is the API documentation for the Vehicle Rental system, which allows users to manage vehicle rentals, reservations, and vehicles.

Rentals

1. Get All Rentals

Retrieve all active rentals.

Endpoint: 
GET api/v1/rental/rentals
Response: 
[
  {
    "_id": "rentalID",
    "vehicleID": "vehicleID",
    "userId": "userID",
    "returnedAt": null,
    "station": "stationName"
  }
]

2. Rent a Vehicle

Rent a vehicle by specifying a vehicle ID and a user ID.

Endpoint: 
POST api/v1/rental/rentals/add
Body Parameters:
Field Type Description
vehicleID String The ID of the vehicle to rent.
userId String The ID of the user renting the vehicle.
Response: 
{
  "message": "Vehicle rented successfully",
  "rental": { "vehicleID": "vehicleID", "userId": "userID" }
}

3. Return a Rental

Return a rented vehicle by its rental ID and specify the return station.

Endpoint: 
PUT api/v1/rental/rentals/return/:id
Body Parameters:
Field Type Description
station String The station where the vehicle is being returned.
Response: 
{
  "message": "Vehicle returned successfully",
  "vehicle": { "available": true, "station": "stationName" }
}

4. Get a Rental by ID

Endpoint: GET api/v1/rental/rentals/:id

Description: Retrieve a rental by its ID.

Response:

{
    "id": "rental_id",
    "vehicleID": "vehicle_id",
    "userId": "user_id",
    "returnedAt": null
}

5. Update a Rental by ID

Endpoint: PUT api/v1/rental/rentals/update/:id

Description: Update a rental by its ID.

Request Body:

{
    "returnedAt": "2024-01-02T00:00:00Z"
}

Response:

{
    "id": "rental_id",
    "vehicleID": "vehicle_id",
    "userId": "user_id",
    "returnedAt": "2024-01-02T00:00:00Z"
}

Reservations

1. Create a Reservation

Create a new reservation for a vehicle.

Endpoint: 
POST api/v1/rental/reservations/add
Body Parameters:
Field Type Description
vehicleID String The ID of the vehicle being reserved.
userId String The ID of the user reserving the vehicle.
Response: 
{
  "_id": "reservationID",
  "vehicleID": "vehicleID",
  "userId": "userID"
}

2. Get All Reservations

Retrieve all reservations.

Endpoint: 
GET api/v1/rental/reservations
Response: 
[
  {
    "_id": "reservationID",
    "vehicleID": "vehicleID",
    "userId": "userID"
  }
]

3. Cancel a Reservation

Cancel a reservation by its ID.

Endpoint: 
DELETE api/v1/rental/reservations/:id
URL Parameters:
Field Type Description
id String The unique ID of the reservation to cancel.
Response: 
{
  "message": "Reservation cancelled successfully",
  "reservation": { "_id": "reservationID", "userId": "userID", "vehicleID": "vehicleID" }
}

4. Get a Reservation by ID

Endpoint: GET api/v1/rental/reservations/:id

Description: Retrieve a reservation by its ID.

Response:

{
    "id": "reservation_id",
    "vehicleID": "vehicle_id",
    "userId": "user_id",
    "createdAt": "2024-01-01T00:00:00Z"
}

5. Update a Reservation by ID

Endpoint: PUT api/v1/rental/reservations/:id

Description: Update a reservation by its ID.

Request Body:

{
    "vehicleID": "new_vehicle_id",
    "userId": "new_user_id"
}

Response:

{
    "id": "reservation_id",
    "vehicleID": "new_vehicle_id",
    "userId": "new_user_id",
    "updatedAt": "2024-01-02T00:00:00Z"
}

Errors

Common error responses:

{
  "error": "Message describing the error"
}

Vehicles

1. Get Available Vehicles

Retrieve all available vehicles for rent.

Endpoint: 
GET /vehicles
Response: 
[
  {
    "_id": "vehicleID",
    "type": "Bicycle",
    "station": "stationName",
    "available": true
  }
]

2. Add a New Vehicle

Add a new vehicle to the system.

Endpoint: 
POST api/v1/rental/vehicles/add
Body Parameters:
Field Type Description
type String The type of the vehicle (e.g., Bicycle, Scooter, Skateboard).
station String The station where the vehicle is available.
Response: 
{
  "message": "Vehicle added successfully",
  "vehicles": { "type": "Bicycle", "station": "stationName" }
}

3. Get Vehicle by ID

Retrieve a vehicle by its unique ID.

Endpoint: 
GET api/v1/rental/vehicles/:id
URL Parameters:
Field Type Description
id String The unique ID of the vehicle.
Response: 
{
    "_id": "vehicleID",
    "type": "Scooter",
    "station": "stationName",
    "available": true
}

4. Delete a Vehicle

Delete a vehicle by its ID.

Endpoint: 
DELETE api/v1/rental/vehicles/:id
URL Parameters:
Field Type Description
id String The unique ID of the vehicle to delete.
Response: 
{
"message": "Vehicle deleted successfully",
"vehicle": { "_id": "vehicleID", "type": "Scooter", "station": "stationName" }
}

5. Get Vehicles by Station

Retrieve vehicles available at a specific station.

Endpoint: 
GET api/v1/rental/station/:station
URL Parameters:
Field Type Description
station String The name of the station to get vehicles from.
Response: 
[
    {
    "_id": "vehicleID",
    "type": "Skateboard",
    "station": "stationName",
    "available": true
    }
]
